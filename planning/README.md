# Planning Documents

This folder holds the landing-page planning docs for the current demand-validation phase.

## Purpose

These sites are not full sales sites yet. They are one-page demand-testing landing pages intended to:

1. capture contact interest
2. measure which offer gets the strongest response
3. collect enough information to follow up manually by email
4. guide later product and service development

## Included Documents

- `landing-page-brief-option-1.md` - Managed AI assistant access offer
- `landing-page-brief-option-2.md` - Done-for-you AI systems offer
- `landing-page-brief-option-3.md` - Growth and operations support offer
- `website-build-prompt.md` - build prompt/spec for when development starts
- `lead-flow-and-validation-notes.md` - conversion and follow-up logic

## Current Strategy

- All three sites should share a similar one-page structure under the hood
- Each site should still feel distinct in brand, messaging, and offer clarity
- No pricing is shown yet
- Primary conversion event is contact submission
- Additional qualification happens later by email
