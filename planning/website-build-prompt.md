# Website Build Prompt

Use this prompt when it is time to actually build the landing page.

---

Create a polished one-page landing website in Next.js for a demand-validation test.

## Core objective

The page is meant to test demand for a business service, not fully sell a mature product yet.

The website should:

- explain the offer clearly within a few seconds
- look trustworthy and modern
- keep the page short, clean, and highly readable
- drive visitors toward a contact form
- collect simple contact information only
- avoid pricing
- avoid excessive technical detail

## Service context

This particular repo is for Option 1:

A managed AI assistant service for small businesses.

The offer is:

- we set up the AI assistant
- we host it
- we maintain it
- we keep it secure and working
- the client gets access to use it
- the client does not need to handle technical setup or maintenance

The website should present this as a business-friendly managed service, not as a developer tool and not as a highly technical product.

Do not emphasize OpenClaw as the front-facing brand. Keep the experience outcome-first.

## Positioning goals

The visitor should quickly understand:

- this is a managed AI assistant for business use
- they can get access without dealing with the technical side
- the service is intended for small businesses
- contacting us is the next step if they are interested

## Tone and style

The page should feel:

- clean
- modern
- calm
- trustworthy
- professional
- slightly premium
- not flashy
- not generic startup hype

The copy should be:

- concise
- plain-language
- outcome-focused
- easy for non-technical business owners to understand

## Visual direction

Use a restrained and cohesive design.

Recommended direction:

- light or off-white background
- dark slate or charcoal text
- muted blue accent color
- generous spacing
- strong section hierarchy
- subtle visual polish
- minimal, tasteful motion if any

Avoid:

- loud gradients
- neon AI aesthetics
- cluttered dashboards
- heavy jargon
- walls of text

## Page structure

Build a one-page landing page with these sections:

1. Hero
2. What this is
3. Example use cases
4. Why businesses choose this
5. How it works
6. Contact form
7. FAQ

## Hero requirements

The hero should clearly communicate the offer.

It should include:

- strong headline
- short explanatory subheadline
- clear primary CTA leading to the contact section
- optional secondary supporting line if useful

The headline should be straightforward, not clever.

## Messaging priorities

Prioritize these ideas:

1. We handle setup, hosting, maintenance, and security
2. The client gets direct access to a working AI assistant
3. The service removes technical friction
4. The assistant can help with many day-to-day business tasks
5. Reaching out is simple and low-pressure

## Use case examples

Use broad but concrete examples such as:

- drafting emails
- answering internal questions
- organizing business knowledge
- helping with repetitive admin tasks
- assisting with planning and research

## Contact form requirements

Keep the form intentionally light.

Fields:

- name
- email
- business name
- optional message

The form should feel easy and low-friction.

## Conversion context

The website is primarily for collecting interest.

After someone submits the form, the team will follow up manually by email with deeper questions and possibly a later survey.

The page itself does not need to handle full qualification.

## UX requirements

- mobile-first and responsive
- high readability
- strong CTA visibility
- accessible color contrast
- simple scrolling experience
- no unnecessary navigation complexity

## Technical and implementation guidance

- build with the current Next.js app-router structure already in this repo
- keep the code clean and maintainable
- prefer reusable sections and simple composition
- avoid overengineering
- ensure the page builds cleanly with no errors
- ensure a good default Vercel deployment experience

## Final goal

Produce a landing page that feels credible, clear, and conversion-focused for early demand testing, while leaving room to evolve into a more mature service site later.
